# Google Ranking Systems Reference

Quick reference for all known Google ranking systems. Understanding these helps prioritize SEO efforts.

## Active Ranking Systems

### BERT (Bidirectional Encoder Representations from Transformers)
**What it does:** Understands how word combinations express different meanings and intents, especially in longer natural language queries.
**Optimize by:** Writing naturally, focusing on concepts and semantic relationships rather than exact-match keywords. Analyze SERPs to understand true intent behind conversational queries.

### RankBrain
**What it does:** Machine learning system that understands how words relate to concepts. Returns relevant results for previously unseen queries. Adjusts rankings based on user satisfaction signals (clicks, Dwell Time).
**Optimize by:** Satisfying user intent immediately. Increase Dwell Time and prevent pogo-sticking by delivering value above the fold.

### Neural Matching
**What it does:** Matches concepts in queries and pages even when they don't share exact words.
**Optimize by:** Covering topics comprehensively with LSI keywords so Google understands the broader context.

### MUM (Multitask Unified Model)
**What it does:** Advanced AI that understands and generates language. Used for specific applications like improving vaccine searches and featured snippet callouts.
**Optimize by:** Providing direct, clear answers using tables and lists that AI can parse and extract.

### Link Analysis Systems & PageRank
**What it does:** Analyzes web page link relationships to determine importance, relevance, and trustworthiness. PageRank measures quality and quantity of inbound links.
**Optimize by:** Earning natural backlinks from high-authority sites. Using descriptive anchor text. Creating linkable assets (original research, free tools).

### Passage Ranking System
**What it does:** Identifies individual page sections to determine relevance of specific portions to a query. Allows long-form content to rank for highly specific questions.
**Optimize by:** Using semantic chunking with descriptive H2/H3 headings. Starting each section with a direct answer to the heading's implied question.

### Page Experience System (Core Web Vitals)
**What it does:** Evaluates real-world UX via LCP (loading), INP (responsiveness), CLS (visual stability). Also considers HTTPS and mobile-friendliness.
**Optimize by:** LCP < 2.5s, INP < 200ms, CLS < 0.1. Compress images, minimize CSS/JS, fast hosting, no intrusive interstitials.

### Freshness Systems
**What it does:** Shows fresher content for queries that deserve freshness (breaking news, recent events, new releases).
**Optimize by:** Regularly updating content, using "Updated on" dates, maintaining steady new link growth.

### Original Content Systems
**What it does:** Ensures original content (primary reporting, original research) is displayed ahead of aggregators.
**Optimize by:** Publishing original data, unique insights, and expert analysis — not rewritten competitor content.

### Reviews System
**What it does:** Evaluates first-party review content to reward high-quality reviews with in-depth research and original analysis.
**Optimize by:** Writing from first-hand perspective with original photos/videos, pros/cons, competitor comparisons, and quantitative measurements.

### Reliable Information Systems
**What it does:** Elevates high-quality journalism and authoritative pages. Displays content advisories when confidence is low.
**Optimize by:** Strict E-E-A-T compliance. Factual accuracy, clear author credentials, reputable source citations.

### Deduplication Systems
**What it does:** Identifies similar/identical content, selects canonical page for display.
**Optimize by:** Using `rel="canonical"` tags. Consolidating thin, competing pages into single comprehensive resources.

### Spam Detection Systems
**What it does:** Identifies and penalizes content created to manipulate rankings (scaled content abuse, deceptive link practices).
**Optimize by:** Avoiding auto-generated fluff, keyword stuffing, aggressive pop-ups. Creating people-first content only.

### Site Diversity System
**What it does:** Generally limits domains to two listings in top results.
**Optimize by:** Consolidating competing pages rather than creating dozens targeting the same keyword.

### Exact Match Domain System
**What it does:** Prevents unfair ranking credit for keyword-stuffed domain names.
**Optimize by:** Building brand reputation through quality content and earned backlinks, not keyword domains.

### Local News Systems
**What it does:** Surfaces relevant local news sources in Top Stories and Local News.
**Optimize by:** Submitting to Google Publisher Center, using News sitemaps.

### Crisis Information Systems
**What it does:** Provides timely information during crises and emergencies.
**Optimize by:** Publishing reliable, timely, factually accurate crisis information (relevant for emergency organizations).

## Retired / Integrated Systems

### Helpful Content System (integrated March 2024)
Now part of core ranking. Ensures people see original content written for humans. Optimize using the "Who, How, Why" framework.

### Hummingbird (2013)
Improved natural language understanding. Now core. Optimize by creating comprehensive content answering related questions.

### Panda System (2011)
Demotes thin, low-quality content. Now core. Optimize by auditing and consolidating thin/outdated pages.

### Penguin System (2012, integrated 2016)
Combats link spam. Now core. Optimize by avoiding paid link schemes; earn links through PR and linkable assets.

## Key Takeaway for Audit Prioritization

1. **Content quality** (Helpful Content + E-E-A-T + Original Content) = highest impact
2. **Technical foundation** (Page Experience + Crawlability) = table stakes
3. **Link authority** (PageRank + Link Analysis) = competitive advantage
4. **Semantic relevance** (BERT + Neural Matching + Passage Ranking) = content structure matters
5. **Freshness** = ongoing maintenance requirement
