# Master Implementation Plan Template

This template defines the exact structure for the plan artifact generated by rad-planner. The plan must be detailed enough that a fresh AI session with zero prior context can execute it successfully ("zero-context ready").

## Template Rules

1. Every task MUST have: ID, Objective, Dependencies, Validation, Rollback
2. Dependencies use task IDs in arrays: `[S1, S2]`
3. Complexity scored 1-10 (tasks >7 must be broken into subtasks)
4. Validation must be a runnable command or verifiable condition
5. Rollback must restore to last known-good state
6. Checkpoints inserted after every milestone
7. Anti-pattern warnings reference specific items from the constraints list
