# Examples

Real artifacts that rad-planner produces — not template placeholders. Use these to see what the workflow actually generates before invoking it on your own project.

| File | What it shows |
|---|---|
| `example-plan.md` | A complete `implementation_plan.md` for a small (~10-task) project, following the 7-section template. Generated by walking through `/rad-planner:plan-project` for a hypothetical URL shortener API. |
| `example-tasks.md` | The corresponding machine-readable task list. Pass this to `scripts/plan-lint.py` to see the validators in action — `python3 ../scripts/plan-lint.py --mode all example-tasks.md`. |

These examples are intentionally small. A real plan for a non-trivial project produces 20–50 tasks across 4–6 milestones, with deeper subtask trees on complexity-7+ items.

## Use these to verify the validators

```bash
# Status report on the example
python3 ../scripts/plan-lint.py --mode status example-tasks.md

# Full validation
python3 ../scripts/plan-lint.py --mode all example-tasks.md
```

Both should report no DAG/checklist issues. If you want to see failures, copy `example-tasks.md`, introduce a cycle or strip a Validation field, and re-run.
