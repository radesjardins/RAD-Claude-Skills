# Voice Profile Schema Reference

Defines the structure, attributes, and usage of voice profiles generated by the Voice Analyst agent and consumed by the write, improve, and review skills.

---

## Purpose

A voice profile captures the distinctive patterns of a specific writer's style. It allows the plugin to:
- Generate new text that sounds like the writer (write skill)
- Nudge improvements toward the writer's natural patterns (improve skill)
- Evaluate whether text matches the writer's voice (review skill)

Voice profiles are NOT prescriptive rules. They are descriptive patterns that inform style guidance. A writer might choose to deviate from their profile for a specific piece — the profile is a baseline, not a constraint.

---

## File Format

Voice profiles are markdown files with YAML frontmatter. To use a profile:
- **In a Project:** Upload the profile artifact as Project Knowledge — it will apply to all conversations in that project
- **In a conversation:** Attach the profile artifact alongside this skill
- **From scratch:** Ask "analyze my writing voice" with 3-5 writing samples to generate a new profile

### Frontmatter Schema

```yaml
---
name: [string]              # User-chosen profile name (e.g., "default", "academic", "casual-blog")
created: [YYYY-MM-DD]       # Date profile was generated
updated: [YYYY-MM-DD]       # Date of last update (if different from created)
samples: [integer]           # Number of writing samples analyzed
sample_types: [list]         # Types of writing analyzed (e.g., ["email", "blog", "report"])
description: [string]        # One-line description (e.g., "Professional but warm, direct, slightly informal")
---
```

### Body Sections

The profile body contains these sections in order:

#### 1. Sentence Patterns

Quantitative and qualitative description of how this writer constructs sentences.

**Attributes to capture:**
- **Average sentence length:** [number] words
- **Sentence length range:** [min]-[max] words
- **Sentence length SD:** [number] (variance indicator)
- **Dominant structures:** What syntactic patterns appear most often (e.g., "leads with subject-verb, but frequently uses fronted adverbials")
- **Fragment usage:** How often and in what contexts (e.g., "uses fragments for emphasis, roughly 1 per 500 words")
- **Complex sentence frequency:** How often sentences use subordinate clauses, embedded phrases

#### 2. Vocabulary

The writer's word preferences and avoidances.

**Attributes to capture:**
- **Vocabulary level:** plain / conversational / professional / technical / academic
- **Jargon comfort:** Does the writer use industry jargon freely, sparingly, or define it inline?
- **Preferred terms:** Specific word choices that recur (e.g., "uses 'fix' not 'resolve', 'think' not 'believe'")
- **Avoided terms:** Words this writer never or rarely uses
- **Contraction usage:** Always, sometimes, never (e.g., "always uses contractions in prose, avoids in headings")

#### 3. Tone

The emotional and interpersonal qualities of the writing.

**Attributes to capture:**
- **Formality:** Scale 1-10 (1 = texting a friend, 10 = legal brief)
- **Warmth:** Scale 1-10 (1 = clinical/detached, 10 = personal/intimate)
- **Directness:** Scale 1-10 (1 = hedged/diplomatic, 10 = blunt/assertive)
- **Humor:** None / dry / self-deprecating / playful / absent (describe the type if present)
- **Confidence:** How the writer handles uncertainty — states opinions directly? Hedges carefully? Acknowledges limits openly?

#### 4. Structure

How the writer organizes paragraphs and documents.

**Attributes to capture:**
- **Paragraph length range:** short (1-3 sentences) / medium (3-5) / long (5+) / varied
- **Opening patterns:** How paragraphs typically begin (e.g., "often starts with a question or a concrete scene")
- **Transition style:** Explicit connectors vs. implicit flow vs. mix
- **Document flow:** Deductive (conclusion first) vs. inductive (builds to conclusion) vs. narrative
- **List usage:** Frequent? Rare? Numbered or bulleted?

#### 5. Distinctive Markers

Patterns unique to this writer — the things that make their writing recognizably theirs.

**Attributes to capture:**
- **Signature moves:** Rhetorical patterns that recur (e.g., "ends emails with a question rather than a statement")
- **Punctuation habits:** Em dash user? Semicolon avoider? Ellipsis in casual writing?
- **Parenthetical style:** Uses parentheses for asides? Dashes? Neither?
- **Rhetorical questions:** Frequent? Rare? Strategic?
- **Perspective:** First person? Third? Mix? "We" vs "I"?

#### 6. Anti-patterns

What this writer does NOT do — equally important for maintaining authenticity.

**Attributes to capture:**
- **Never uses:** Specific words, phrases, or constructions this writer avoids
- **Structural avoidances:** Patterns absent from their writing (e.g., "never uses bullet points in prose")
- **Tonal boundaries:** What tones this writer avoids (e.g., "never sarcastic in professional writing")

## Example Profile

```yaml
---
name: default
created: 2026-04-07
samples: 5
sample_types: ["email", "blog", "report"]
description: "Direct and conversational, professional without being stiff, favors concrete examples over abstract claims"
---

## Sentence Patterns
- Average length: 14 words
- Range: 3-35 words
- SD: 9.2 (high variance — natural)
- Leads with subject-verb most often, but breaks pattern every 3-4 sentences
- Uses fragments for emphasis, roughly 1 per 300 words
- Occasional one-word sentences ("No." "Exactly." "Done.")

## Vocabulary
- Level: conversational-professional
- Uses contractions always ("don't" not "do not", "it's" not "it is")
- Prefers short words: "use" not "utilize", "help" not "facilitate", "show" not "demonstrate"
- Industry jargon used without definition when audience is technical
- Never uses: delve, leverage, foster, paradigm, synergy

## Tone
- Formality: 4/10
- Warmth: 6/10
- Directness: 9/10
- Humor: dry, occasional — roughly once per 500 words
- States opinions without hedging: "This approach is wrong" not "This approach may not be optimal"

## Structure
- Paragraph length: varied (1-8 sentences, frequently uses one-liners)
- Opens paragraphs with scenes, questions, or direct statements — rarely topic sentences
- Transitions are implicit — relies on logical flow, almost never uses "Furthermore" or "Additionally"
- Document flow: leads with the conclusion, then explains
- Uses bullet points for actual lists, never for prose content

## Distinctive Markers
- Ends key paragraphs with a short, punchy sentence
- Uses em dashes sparingly (1-2 per page) for genuine asides
- Asks rhetorical questions to set up the next section
- Uses "I" freely, not "one" or passive constructions
- Parenthetical asides for tangential thoughts

## Anti-patterns
- Never uses passive voice unless quoting someone else
- Never hedges with "arguably," "potentially," or "it could be said"
- Never starts paragraphs with "In today's..." or "It's important to note..."
- Never uses more than one exclamation mark per document
- Never uses emoji in professional writing
```

## How Skills Consume Profiles

### Write Skill
- Reads the profile before generating
- Matches sentence length distribution, vocabulary level, and tone
- Avoids anti-patterns listed in the profile
- Uses distinctive markers as stylistic targets

### Improve Skill
- Compares submitted text against the profile
- Flags deviations that weaken voice consistency
- Suggests changes that move text toward the profile's patterns
- Does NOT enforce the profile rigidly — allows intentional deviation

### Review Skill
- Includes "voice consistency" as a scored category when a profile is active
- Notes where the text sounds unlike the writer
- Distinguishes between intentional voice shifts and unintentional drift

## Profile Generation Guidelines (for Voice Analyst agent)

### Minimum Requirements
- At least 3 writing samples
- Samples should represent the writer's typical output (not their best or worst)
- Ideally from the same domain (all emails, all blog posts) or explicitly mixed

### Analysis Process
1. Read all samples before forming any conclusions
2. Look for patterns that appear across ALL samples, not just one
3. Quantify where possible (sentence length stats, contraction frequency)
4. Note what's absent as carefully as what's present
5. Write the profile in a descriptive tone, not prescriptive
6. Include specific examples from the samples to illustrate patterns

### claude.ai Output Format
When generating a profile for claude.ai, include this header:

```
## How to Use This Voice Profile

1. Download this artifact or copy its contents
2. Open your Claude.ai Project settings
3. Add this file to Project Knowledge
4. Every conversation in this Project will now write in your voice

To use across multiple Projects, add the same file to each Project's Knowledge.
This profile was generated from [N] writing samples on [date].
To update it, provide additional writing samples and ask the Voice Analyst to refresh.
```

### Updating Profiles
- New samples can be added without replacing the profile from scratch
- The agent should compare new samples against existing patterns and note changes
- Attributes can be tweaked on request: "make it less formal" → adjust formality from 4/10 to 3/10 and cascade effects (more contractions, shorter sentences, more casual vocabulary)
